# 空白 ⌘CC → 開空白 Workbench（貼給 Claude Code 的任務）

> **從 master 開分支 `feature/blank-workbench`。**（這張先做；菜單列排在其後。）
> 開始前讀 `CLAUDE.md`、`docs/decisions.md`、`docs/issues.md`。

## 目標

讓「沒有複製任何新東西時按 ⌘CC」有明確用途：**直接開一個全空的 Workbench，讓使用者自己打字翻譯**（類似 Google 翻譯的主動輸入）。

這同時解決一個既有困惑：現在空白按 ⌘CC 會帶出「上一次的剪貼簿內容」，讓人莫名其妙。賦予這個動作明確語意後，困惑消失。

## 核心邏輯：用 changeCount 判斷「這次有沒有新複製」

⌘CC 觸發時，判斷**這次的雙擊動作有沒有產生新的複製**（用 macOS 剪貼簿的 `changeCount`——每次剪貼簿被寫入就 +1，比對它即可）：

```
⌘CC 觸發
├─ 有新複製（changeCount 比上次升高）→ 照舊：翻譯這次複製的內容（現狀，Glance）
└─ 無新複製（changeCount 沒變）       → 開一個全空的 Workbench
```

- **判斷依據是「這次有沒有產生新複製」，不綁時間。** 例如十分鐘前複製過、現在在空白處按 ⌘CC（沒選任何東西、changeCount 沒升高）→ 視為「無新複製」→ 開空白 Workbench。
- 原理：第一次 ⌘C 若有選取就會複製、changeCount 升高；空白處按則不會。比對 changeCount 比「判斷剪貼簿內容是不是空」可靠得多（剪貼簿幾乎永遠留著舊內容）。

## 空白 Workbench 的行為（已拍板）

- **開全空的**：原文欄完全空白、游標就緒等使用者打字。**不要**把上次剪貼簿內容塞進去（這正是要消除的困惑）。
- **直接開 Workbench**，跳過 Glance（自己打字翻譯需要可編輯、會拿 focus 的視窗，Glance 那種唯讀小浮窗不適合）。
- 打字後 → **沿用既有 Workbench 的 debounce 即時重翻管線**，套用語言配對/目標語言設定，跟一般 Workbench 完全一樣（字典點字、複製譯文等都照舊可用）。

## 技術約束

- 沿用既有觸發層（偵測雙擊 ⌘C 的全域 event tap）與既有 Workbench 開啟流程、重翻管線——**不要另寫一套**。
- changeCount 透過已在用的 objc2-app-kit 取得，應不需新 crate（若需要，先列出、說明、等我確認）。
- 不改動「有新複製 → Glance 翻譯」這條既有路徑。
- 紅線照舊：不 log 內容、key 只進 Keychain、secret 一律 gitignore。修 bug 記 `docs/issues.md`。

## 驗收標準（每條可手動驗證）

1. 反白文字、雙擊 ⌘C（有新複製）→ 照舊出 Glance 翻譯（現狀完全不變）。
2. 沒選任何東西、在空白處雙擊 ⌘C（無新複製）→ 開一個 Workbench。
3. 該 Workbench **原文欄全空、不帶任何上次剪貼簿內容**，游標就緒可打字。
4. 在空白 Workbench 打字 → debounce 後即時翻譯，套用語言配對設定。
5. 判斷不綁時間：很久以前複製過、現在空白按 ⌘CC 仍視為「無新複製」→ 開空白 Workbench。
6. 既有功能（句子翻譯、字典點字、語言配對）皆不受影響。
7. repo 無 secret、無內容被 log。

## 附帶修正：設定視窗關掉後叫不回來

**症狀：** build 出來的正式版，第一次開設定頁、關掉後，再點 App icon 叫不出設定。

**先隔離根因再修（別預設答案）：** 可能是
- (a) 設定視窗被「關閉」時是**整個銷毀**，而非隱藏——再叫已經沒有視窗可叫（與 issue #5「原生關閉鈕銷毀視窗」同一類）。
- (b) 點 Dock icon 根本沒綁定「開設定」這個動作（Sumi 沒有傳統主視窗）。

先確認是哪一個，再修。**修法原則：設定視窗關閉應為「隱藏」而非「銷毀」**，確保之後不論從哪個入口都能重新叫出。

**與菜單列的關係（要注意）：** 之後做 `feature/menu-bar` 時，App 會變成沒有 Dock icon 的常駐程式，「叫出設定」的正規入口會改成選單列。所以這裡的重點是把設定視窗做成「關閉=隱藏、可重複叫出」，這樣不論之後從 Dock 或選單列叫都叫得回來。

**驗收：** 開設定 → 關閉 → 再次觸發（點 icon / 重新叫設定）→ 設定視窗能重新出現，內容正常。

# Sumi — UI 設計規範 (ui-spec)

給前端建置用的設計依據。所有顏色、字級、間距都從這裡取，**不要各自發明**。
搭配 `docs/PRD.md`（功能/行為）一起看。

---

## 1. 設計主張（thesis）

**水墨 × 紙（sumi-e on washi），落一枚朱印。**
近黑「墨色」字落在暖白「紙」底，大量留白，精緻 hairline 分隔。全產品只用**一個**點綴色——朱紅，取自水墨畫的落款印（hanko），只出現在簽名級元素上。

⚠️ **刻意避開的反例**：近黑背景 + 單一霓虹/亮色 accent，是目前 AI 設計的套版長相。Sumi 是**紙底**不是黑底，朱紅是有來歷的「印」而非裝飾性亮點——別做成暗黑霓虹風。

**Signature 元素（全產品記憶點）**：
1. 朱印方塊——logo、以及「啟用中 / 選字」狀態的標記。
2. 墨痕串流游標——譯文串流時，行尾一道朱色短直畫（像筆鋒）。

## 2. 顏色 token

以 CSS 變數實作，light / dark 都要可用。

| Token | Light | Dark | 用途 |
|---|---|---|---|
| `--paper` | `#FAF8F3` | `#161410` | App 底（紙） |
| `--surface` | `#FFFFFF` | `#211E18` | 卡片 / 浮窗面 |
| `--ink` | `#1C1A17` | `#ECE8DF` | 主文字（墨） |
| `--ink-2` | `#57534C` | `#A8A296` | 次要文字 |
| `--ink-3` | `#8A857C` | `#6F695E` | 提示 / 弱化 |
| `--hairline` | `rgba(28,26,23,.12)` | `rgba(236,232,223,.14)` | 分隔線 / 邊框 |
| `--accent`（朱） | `#C8442B` | `#E0664B` | 印、串流游標、選字高亮 |
| `--accent-bg` | `#F6E7E2` | `#3A201A` | 選字 / accent 區塊底 |
| `--danger` | `#A32D2D` | `#E24B4A` | 機密內容跳過提示 |

規則：accent 是稀缺資源，一個畫面最多出現在 1–2 個地方。其餘一律墨色階。

## 3. 字體 token

- `--font-ui`：系統 UI 字（`-apple-system, "SF Pro Text"`）——速度與原生感優先，body/介面全用它。
- `--font-cjk`：`"PingFang TC", "Noto Sans TC"`——中文。
- `--font-mono`：`"SF Mono", "JetBrains Mono"`——音標、按鍵符號、log/程式碼。
- `--font-display`：一支克制的特色襯線（如 `Newsreader`）——**只**用在 wordmark 與 onboarding 標題，給識別感，不拖慢工具本體。

字重只用兩級：**400 / 500**（不用 600、700）。一律 sentence case。

字級：
| 場景 | 級數 |
|---|---|
| Glance 原文（弱化） | 13 |
| Glance 譯文（主角） | 16 |
| Glance 語言標 / 提示 | 12 / 11 |
| Workbench 欄位標 | 11 |
| Workbench 內文 | 14 |
| 字典 headword / 釋義 | 15 / 13 |

最小字級 11px，不再小。

## 4. 間距 / 圓角

- 間距階：`4 · 8 · 12 · 16 · 24`（px，元件內），`1rem · 1.5rem · 2rem`（垂直節奏）。
- 圓角：卡片 `12px`、一般元件 `8px`、朱印方塊 `2px`。
- 邊框一律 `0.5px solid var(--hairline)`。

---

## 5. 版面 — Glance 模式

- 寬度 `320–360px`；高度依內容；出現在游標附近。
- non-activating panel，always-on-top，**不搶 focus**。
- 結構（上→下）：語言標 + 展開鈕 ／ 原文（弱化） ／ hairline ／ 譯文（主角，串流，行尾朱色游標） ／ 底部提示（`Esc 關閉 · ⌘↩ 展開`）。
- **key terms**：預設不顯示（守極簡）；設定可開，開了在譯文下加一行詞彙 chips。
- 行為：`Esc` 或失焦關閉；閒置自動淡出（預設約 6s，可調）。

## 6. 版面 — Workbench 模式

- 視窗 min `560×360`，可調整大小，**拿 focus**。
- Toolbar 高 `44px`：左 = 朱印 + `Sumi` wordmark；右 = 目標語言、provider、複製。
- 主區**自適應**（這是定案，不二選一）：
  - 寬度 ≥ `520px` → **左右雙欄**（原文｜譯文）。
  - 寬度 < `520px` → **上下堆疊**（原文在上、譯文在下）。
- 原文欄可編輯；編輯 debounce（約 400ms）後即時重翻。
- **字典卡**（點選單字浮出，anchored 於選字處）分兩段，順序固定：
  1. 真字典：headword · 音標(mono) · 詞性 · 釋義。
  2. LLM（標明 provider，如 Gemini）：文法 / 語境 / 改寫建議。
  這個視覺分段就是 PRD「字典 ≠ LLM」的具體呈現，**不可合併成一段**。

---

## 7. 動態（克制）

- Glance 進出：`120ms` 淡入淡出，無位移彈跳。
- 譯文串流：逐 token 顯現，行尾朱色游標。
- 其餘互動：hover/active 用底色變化，不用陰影、不用 glow。
- **尊重 reduced-motion**：關閉淡入與游標動畫，直接顯示。

---

## 8. 品質底線（不可省）

- 完整 dark mode（上表兩欄都要實作）。
- 鍵盤可操作 + 可見的 focus ring。
- 尊重系統 reduced-motion。
- 文案：動詞主動式、sentence case、講使用者看得懂的話（「複製」不是「Submit」）；錯誤訊息說「怎麼了、怎麼解」，不道歉、不含糊。
